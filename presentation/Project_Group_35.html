OpenSky Flight Data Pipeline & Traffic Analytics¶

Educational end-to-end analytics workflow¶

This ZHAW university group project uses OpenSky state-vector data to demonstrate API ingestion, cleaning, SQL, EDA, statistics, regression mechanics, and clustering. The price variable is synthetic and does not represent real ticket prices.

1. Data Collection¶

This section collects raw flight data using a public Web API (OpenSky Network).

2. Data Preparation¶

This section filters and cleans the state vectors. It also generates a synthetic price variable for the original academic exercises; that variable is not an OpenSky field or a real airfare.

6. Educational Regression Demonstration¶

The target is synthetic and was constructed partly from velocity and geometric altitude, which are also predictors. This exercise demonstrates train/test splitting, fitting, RMSE, and R²; its performance is not evidence of real-world ticket-price predictability.

In [33]:
# Features und Zielvariable
X = df_clean[["velocity", "geo_altitude", "latitude", "longitude"]].copy()
y = df_clean["price"].copy()

data_model["price"] = y
data_model = data_model.dropna()

y = data_model["price"]

# Train-Test-Split
X_train, X_test, y_train, y_test = train_test_split(

7. Descriptive Flight-Profile Clustering¶

KMeans explores three groups in the selected feature space. The groups are descriptive, include the synthetic price feature, and are not official aircraft, fare, or operational categories.

In [39]:
# Features fürs Clustering
X_cluster = df_clean[["velocity", "geo_altitude", "price"]].dropna()

# Skalieren
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_cluster)

# k-Means
kmeans = KMeans(n_clusters=3, random_state=42)
clusters = kmeans.fit_predict(X_scaled)

X_cluster["cluster"] = clusters
